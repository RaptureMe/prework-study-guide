HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The head element contains Meta data, we learned that the ALT info is good for search engines.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • the .card in the css file assigns to the class selector '.card"-- the 'section' element in the HTML markup will now how those changes

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • git add -A: The 'add' command adds modifications in the current working branch to the staging area. The -A flag indicates that we want to add all changes.
  • git commit -m "comment-here": In the preceding Git statement, a commit was created that captures a snapshot of the currently staged changes. The -m flag is to associate a message with our commit
  • git pull origin main: command to receive a branch's modifications into the local environment from the main branch.
  • git push origin feature/branch-name: From local environment push our changes to the remote branch

JavaScript

  • In JavaScript, to declare a variable, we use the var keyword.
  • Each line of JavaScript code ends with a semicolon.
  • An if statement is a way of introducing decision-making into our code.
  • The strict equality operator (===) checks to see if two values are equal, and returns a Boolean result true if the values are equal and false if the values are not equal.
  • A for loop uses the predictable pattern of indices to perform a task on all the items in an array by allowing a single code block to be executed over and over.
